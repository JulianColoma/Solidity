# 📚 Solidity Mastery: El Estándar ERC-721 (NFTs)

A diferencia del ERC-20, donde todos los tokens son iguales, en el **ERC-721** cada token tiene un ID único. Es un estándar de "coleccionables" o "activos únicos".

---

## 📝 Teoría

Funciones clave que lo diferencian del ERC-20:

1. **`ownerOf(tokenId)`**: Devuelve la dirección del dueño de un ID específico.
2. **`tokenURI(tokenId)`**: Devuelve un enlace (generalmente a IPFS) con los metadatos del token (imagen, nombre, descripción).
3. **`safeTransferFrom(...)`**: Envía el token de forma segura, verificando que el receptor sea capaz de recibirlo (para evitar que los NFTs se pierdan en contratos que no los soportan).



---

## 📌 Puntos Clave

1. **Escasez Programada:** Cada token tiene un `tokenId` único (ej. el NFT #1 es distinto al NFT #2).
2. **Metadatos:** La información del NFT (como la imagen de un pabellón de la facultad) no se guarda "dentro" de la blockchain porque es muy caro. Se guarda un link (URI) que apunta a un archivo JSON externo.
3. **Uso de Librerías:** Al igual que con el ERC-20, heredamos de OpenZeppelin para garantizar que el contrato siga todas las reglas de seguridad.
