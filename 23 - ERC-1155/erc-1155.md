# 📚 Solidity Mastery: El Estándar ERC-1155 (Multi-Token)

El **ERC-1155** es conocido como el "estándar de oro" de los tokens. Su gran ventaja es que permite gestionar múltiples tipos de tokens (fungibles y no fungibles) dentro de un **mismo Smart Contract**.

---

## 📝 Teoría

* **Eficiencia de Gas:** En lugar de desplegar un contrato para tu moneda y otro para tus NFTs, desplegás uno solo.
* **Transferencias en Lote (Batch Transfers):** Podés enviar 5 tipos de tokens diferentes a un amigo en una sola transacción. Esto ahorra muchísimo gas.
* **ID de Token:** A diferencia del ERC-721 donde el ID es único por pieza, acá el ID representa una **clase de objeto**.
  * ID 1: Monedas de oro (pueden haber un millón).
  * ID 2: Espada de fuego (puede haber solo una).



---

## 📌 Puntos Clave

1. **Un solo contrato:** Ideal para sistemas de inventario, juegos o plataformas que manejan distintos tipos de certificados.
2. **URI Dinámica:** En lugar de guardar una URL por cada token, usás una base (ej. `https://api.tuweb.com/item/{id}.json`) donde `{id}` se reemplaza automáticamente por el ID del token.
3. **Flexibilidad:** Podés definir que un ID se comporte como un NFT (poniendo un suministro máximo de 1) o como una moneda (suministro ilimitado).
